# Minilog JPA

책 『스프링 부트 개발자 온보딩 가이드』 실습 내용을 기반으로 **Spring Boot 프로젝트를 체계적으로 관리하기 위한 학습용 레포지토리**입니다.  
실습, 개념 정리, 개선 작업을 GitHub 이슈와 칸반 보드를 활용하여 관리합니다.

---

## 📌 프로젝트 목표

- Minilog API에 안전한 인증 시스템을 추가합니다.
- 단순 코드 작성이 아닌 **이슈 → PR → 머지** 흐름에 익숙해집니다.
- GitHub 이슈 템플릿과 프로젝트 보드를 활용한 **프로젝트 관리**를 연습합니다.

---

## 💡요구사항

- 기능
    - JWT(JSON Web Token)의 생성 및 검증 기능 추가
        - JWT에 사용자의 User Name, Id, 발급 시간, 만료 시간 포함
        - AUTHOR와 ADMIN 두 가지 권한 부여 기능 제공
            - AUTHOR: 게시글 작성 및 조회 권한
            - ADMIN: 게시글 관리 및 사용자 관리(수정, 삭제) 권한
    - 각 컨트롤러의 엔드포인트에 JWT 인증 기능 추가. 즉, 인가되지 않은 사용자는 엔드포인트에 접근하지 못하도록 차단
- 구현
    - 로그인 및 Swagger 페이지를 제외한 모든 엔드포인트는 인증을 필수적으로 요구
    - User ID를 입력 받는 기존 컨트롤러 및 DTO를 수정하여, JWT에 포함된 식별자를 사용하도록 변경

## 🗂 이슈 관리 방식

이 프로젝트의 모든 작업은 GitHub 이슈를 통해 관리합니다.

- **실습(Task)**  
  책의 실습 단위를 관리하기 위한 이슈

- **개념 정리(Concept)**  
  학습 중 정리해야 할 개념 기록

- **개선/리팩터링(Refactor)**  
  코드 구조나 설정 개선 작업

이슈는 GitHub Projects(칸반 보드)를 통해  
`To do → In Progress → Done` 흐름으로 관리합니다.

---

## 🔀 작업 흐름

1. 이슈 생성
2. `main` 브랜치에서 작업 브랜치 생성
3. 작업 후 커밋 및 푸시
4. Pull Request 생성
5. 코드 확인 후 `main` 브랜치로 머지
6. 머지 완료된 브랜치 정리
